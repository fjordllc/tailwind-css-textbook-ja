# 演習（第8部）

最後は「自分で判断する」演習です。正解は 1 つではありません。

1. **移行を計画する**: 既存の v3 プロジェクトを v4 に上げる手順を、[§29.6](chapter29.md#296-v3--v4-移行の実務) を見ずにまず書き出し、その後で照らし合わせてください。とくに `shadow-*` のリネームと `currentColor` 化は何に注意すべきですか。
2. **採用判断する**: あなたが次に作る（または関わる）プロジェクトを 1 つ思い浮かべ、[§31.1](chapter31.md#311-採用判断のチェックリスト) のチェックリストで採点してください。Tailwind を採用すべきか、理由とともに結論を出しましょう。
3. **言語化する**: 「Tailwind は CSS の代わりではなく、CSS をチームで一貫して書くための層である」（[§31.6](chapter31.md#316-結論-どんな読者に勧められるか)）という主張に、賛成・反対どちらでもよいので、本書全体を踏まえた自分の意見を 200 字程度で書いてください。
